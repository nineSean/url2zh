# Skills 详解：Skills 与提示词、项目、MCP 和子代理有何不同

> 原文: https://claude.com/blog/skills-explained
> 日期: 2025-11-13

---

自推出 **Skills** 以来，大家对 Claude 代理生态系统中各组件如何协同工作产生了浓厚的兴趣。

无论你是在 **Claude Code** 中构建复杂流水线、通过 API 打造企业级方案，还是在 **Claude.ai** 上提升日常效率，弄清楚何时该用哪种工具，将从根本上改变你使用 Claude 的方式。

本指南将逐一拆解每个构建模块，说明各自的适用场景，并演示如何将它们组合起来构建强大的代理工作流（agentic workflows）。

## 认识你的代理构建模块

### 什么是 Skills？

**Skills** 是包含指令、脚本和资源的文件夹，Claude 在处理任务时会动态发现并加载它们。你可以把 Skills 理解为专用的"培训手册"，赋予 Claude 某个领域的专业能力——从处理 Excel 表格到遵循组织的品牌规范。

**Skills 的工作原理：** 当 Claude 接到任务时，会扫描可用的 Skills 寻找匹配项。Skills 采用**渐进式披露（progressive disclosure）**机制：先加载元数据（约 100 token），让 Claude 判断某个 Skill 是否相关；完整指令仅在需要时才加载（少于 5k token）；捆绑的文件或脚本也只按需加载。

**何时使用 Skills：** 当你需要 Claude 持续、高效地执行特定任务时，就该用 Skills。典型场景包括：

- **组织工作流：** 品牌规范、合规流程、文档模板
- **领域专长：** Excel 公式、PDF 处理、数据分析
- **个人偏好：** 笔记体系、编码模式、研究方法

**示例：** 创建一个 **品牌规范 Skill**，包含公司的配色方案、字体规则和版式要求。之后 Claude 在创建演示文稿或文档时会自动应用这些标准，无需你每次都重新说明。

了解更多关于 Skills 的信息，并浏览我们持续增长的 [Skills 库](https://claude.com/skills)。

### 什么是提示词(Prompts)？

**提示词(Prompts)** 是你在对话中以自然语言向 Claude 下达的指令。它们是临时性的、对话式的、即时响应的——你在当下的语境中提供背景和方向。

**何时使用提示词：**

- **单次请求：** "帮我总结这篇文章"
- **对话微调：** "语气再正式一些"
- **即时上下文：** "分析这些数据，找出趋势"
- **临时指令：** "把这段内容格式化成无序列表"

**示例：**

> 请对这段代码做一次全面的安全审查。重点关注：
>
> 1. 常见漏洞，包括：
> - 注入攻击（SQL、命令注入、XSS）
> - 身份验证和授权问题
> - 敏感数据泄露
> - 安全配置缺陷
> - 访问控制失效
> - 加密实现问题
> - 输入校验不足
> - 错误处理和日志问题
>
> 2. 对于每个发现的问题，请给出：
> - 严重等级（紧急/高/中/低）
> - 代码定位（行号或函数名）
> - 风险说明：为什么存在安全隐患，可能如何被利用
> - 具体修复建议和代码示例
> - 防止类似问题的最佳实践
>
> 3. 代码背景：[描述代码用途、语言/框架及运行环境——例如"这是一个处理用户认证和支付数据的 Node.js REST API"]
>
> 4. 额外检查：
> - 是否存在 OWASP Top 10 漏洞？
> - 是否遵循了 [特定框架/语言] 的安全最佳实践？
> - 依赖项中是否存在已知漏洞？
>
> 请按严重程度和潜在影响排序输出结果。

**小贴士：** 提示词是你与 Claude 互动的主要方式，但不会跨对话持久保存。对于重复性的工作流或专业知识，应该考虑把提示词转化为 Skills 或项目指令。

**何时改用 Skill：** 如果你发现自己在不同对话中反复输入同样的提示词，就该创建 Skill 了。把"用 OWASP 标准审查代码安全性"或"按执行摘要、关键发现、建议的结构来分析"这类重复指令做成 Skill，既免去每次重复说明，又能确保执行的一致性。

查看我们的[提示词库](https://claude.ai/prompts)、[提示词最佳实践](https://docs.anthropic.com/claude/docs/prompt-engineering)或[智能提示词生成器](https://claude.ai/prompts)来开始。

### 什么是项目(Projects)？

**项目(Projects)** 在所有 Claude 付费方案中均可使用，是拥有独立聊天记录和知识库的自包含工作区。每个项目有 200K 的上下文窗口，你可以上传文档、提供背景信息，并设置适用于该项目所有对话的自定义指令。

**项目的工作原理：** 上传到项目知识库的所有内容，会在该项目的每次对话中自动可用。Claude 利用这些背景信息给出更精准、更贴合语境的回答。当项目知识接近上下文上限时，Claude 会无缝切换到**检索增强生成(RAG)** 模式，容量最多可扩展 10 倍。

**何时使用项目：**

- **持久上下文：** 需要贯穿每次对话的背景知识
- **工作区隔离：** 为不同方案划分独立上下文
- **团队协作：** 共享知识和对话记录（团队版和企业版）
- **自定义指令：** 项目专属的语气、视角或方法

**示例：** 创建一个"Q4 产品发布"项目，上传市场调研、竞品分析和产品规格文档。此后该项目中的每次对话都能直接访问这些资料，无需反复上传或重新解释背景。

**何时改用 Skill：** 项目为 Claude 提供针对特定工作的持久上下文——比如公司代码库、某项研究计划、持续的客户对接。而 Skills 教的是 Claude **怎么做**。一个项目可能包含产品发布的全部背景资料，而一个 Skill 则教 Claude 团队的写作规范或代码审查流程。如果你发现自己在多个项目中复制粘贴相同的指令，这就是将其转化为 Skill 的信号。

了解更多关于[项目](https://claude.ai/projects)的信息。

### 什么是子代理(Subagents)？

**子代理(Subagents)** 是拥有独立上下文窗口、自定义系统提示词和特定工具权限的专用 AI 助手。子代理在 **Claude Code** 和 **Claude Agent SDK** 中可用，能独立处理具体任务并将结果返回给主代理。

**子代理的工作原理：** 每个子代理都有自己的配置——你可以定义它的能力、解决问题的方式以及可访问的工具。Claude 会根据描述自动将任务派发给合适的子代理，你也可以显式调用特定子代理。

**何时使用子代理：**

- **任务专精：** 代码审查、测试生成、安全审计
- **上下文管理：** 保持主对话聚焦，把专项工作分流出去
- **并行处理：** 多个子代理可同时处理不同方面
- **工具限制：** 限定特定子代理只能执行安全操作（如只读访问）

**示例：** 创建一个只有 Read、Grep 和 Glob 权限而没有 Write 或 Edit 权限的"代码审查"子代理。修改代码时，Claude 会自动派它进行质量和安全审查，且不存在意外改动代码的风险。

**何时改用 Skill：** 如果多个代理或对话需要同样的专业知识（如安全审查流程或数据分析方法），应创建 Skill 而非把知识内嵌到某个子代理中。Skills 可移植、可复用；子代理则是为特定工作流量身定做的。用 Skills 传授任何代理都能使用的专业知识；需要独立的任务执行并配合特定工具权限和上下文隔离时，再用子代理。

了解更多关于[子代理](https://docs.anthropic.com/claude/docs/subagents)的信息。

### 什么是 MCP？

**MCP(模型上下文协议，Model Context Protocol)** 在 AI 应用与你已有的工具和数据源之间建立了通用的连接层。

MCP 是一种开放标准，用于将 AI 助手连接到数据所在的外部系统——内容库、业务工具、数据库和开发环境。

**MCP 的工作原理：** MCP 提供了一种将 Claude 连接到工具和数据源的标准方式。无需为每个数据源单独构建集成，只需面向单一协议开发。MCP 服务器暴露数据和功能；MCP 客户端（如 Claude）连接到这些服务器。

**何时使用 MCP：**

- **访问外部数据：** Google Drive、Slack、GitHub、数据库
- **使用业务工具：** CRM 系统、项目管理平台
- **连接开发环境：** 本地文件、IDE、版本控制
- **对接自定义系统：** 你的专有工具和数据源

**示例：** 通过 MCP 将 Claude 连接到公司的 Google Drive。此后 Claude 可以搜索文档、读取文件并引用内部知识，无需手动上传——连接是持久的且会自动更新。

**何时改用 Skill：** MCP 把 Claude 连接到数据；Skills 教 Claude **如何处理**这些数据。如果你在说明如何使用某个工具或遵循某套流程——比如"查询数据库时务必先按日期范围过滤"或"用这些特定公式来格式化 Excel 报表"——这就是 Skill。如果你需要 Claude 首先能访问到数据库或 Excel 文件，这就是 MCP。两者配合使用：MCP 负责连接，Skills 负责操作知识。

了解更多关于 [MCP](https://modelcontextprotocol.io) 的信息，以及[如何构建 MCP 服务器](https://modelcontextprotocol.io/docs/quickstart)的文档。

## 它们如何协同工作

当这些构建模块组合在一起时，真正的威力才会显现。每个模块各有所长，而它们共同构成了复杂的代理工作流。

### 对比一览：选择合适的工具

| 维度 | Skills | 提示词(Prompts) | 项目(Projects) | 子代理(Subagents) | MCP |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **提供什么** | 操作性知识 | 即时指令 | 背景知识 | 任务委派 | 工具连接 |
| **持久性** | 跨对话 | 仅当次对话 | 项目内 | 跨会话 | 持续连接 |
| **包含什么** | 指令 + 代码 + 资源文件 | 自然语言 | 文档 + 上下文 | 完整代理逻辑 | 工具定义 |
| **何时加载** | 动态按需 | 每轮对话 | 项目内始终加载 | 被调用时 | 始终可用 |
| **能包含代码** | 是 | 否 | 否 | 是 | 是 |
| **最适合** | 专业技能 | 快速请求 | 集中上下文 | 专项任务 | 数据访问 |

### 代理工作流示例：研究代理

下面用一个完整的研究代理来演示如何组合多个构建模块，搭建一个用于竞争分析的代理。

**第 1 步：搭建项目**
创建一个"竞争情报"项目并上传：
- 行业报告和市场分析
- 竞争对手产品文档
- CRM 中的客户反馈
- 以往的研究摘要

添加项目指令：
> 请从我方产品战略的角度分析竞争对手。聚焦差异化机会和新兴市场趋势，以具体证据和可行建议的形式呈现发现。

**第 2 步：通过 MCP 连接数据源**
启用以下 MCP 服务器：
- Google Drive（访问共享研究文档）
- GitHub（审查竞争对手的开源仓库）
- Web Search（获取实时市场信息）

**第 3 步：创建专用 Skills**
创建一个"竞争分析"Skill：

> ### 我司 GDrive 导航 Skill
> #### 概览
> 针对 Meridian Tech 的 Google Drive 结构优化的搜索与检索策略。使用此 Skill 高效定位内部文档、研究成果和战略材料。
> #### 云盘目录结构
> - `/Strategy & Planning/` — OKR、季度计划、董事会报告
> - `/Product/` — PRD、路线图、技术规格
> - `/Research/` — 市场研究、竞争情报、用户研究
> - `/Sales & Marketing/` — 案例研究、路演材料、营销素材
> #### 搜索最佳实践
> 1. **先广后窄** — 结合文件夹上下文与关键词搜索
> 2. **锁定文档归属** — 从 Sales/ 文件夹获取销售材料，而非从根目录
> 3. **关注时效** — 优先使用最近 6 个月内的文档来制定当前战略

**第 4 步：配置子代理（仅限 Claude Code / SDK）**
创建专用子代理：

**市场研究子代理(market-researcher)：**
> 角色：你是一名专注于竞争情报的市场研究分析师。
> 任务：识别权威来源，收集定量数据（市场份额等），分析定性洞察（分析师观点等）。

**技术分析子代理(technical-analyst)：**
> 角色：你是一名分析竞争对手技术选型的技术架构师。
> 任务：审查公开仓库和技术文档，评估架构模式和技术栈。

**第 5 步：启动你的研究代理**
现在，当你问 Claude："分析我们前三大竞争对手如何定位其新 AI 功能，并找出我们可以利用的差距"时，以下流程会依次展开：

1. **项目上下文加载：** Claude 访问你上传的研究文档并遵循项目指令。
2. **MCP 连接激活：** Claude 搜索 Google Drive 获取最新竞品简报，并抓取 GitHub 数据。
3. **Skills 介入：** "竞争分析"Skill 提供分析框架。
4. **子代理执行（Claude Code 中）：** `market-researcher` 收集行业数据，`technical-analyst` 审查技术实现。
5. **提示词精炼：** 你补充对话引导："重点关注医疗保健领域的企业客户"。

**最终产出：** 一份全面的竞争分析报告，整合了多个数据源，遵循你的分析框架，调用了专业技能，并在整个研究过程中保持了上下文的连贯性。

## 常见问题

#### Skills 是怎么工作的？
Skills 通过**渐进式披露**来保持 Claude 的高效运作。处理任务时，Claude 先扫描 Skill 元数据（描述和摘要）来识别相关匹配；匹配成功后才加载完整指令；最后，如果 Skill 包含可执行代码或参考文件，这些内容也只在需要时才加载。
这种架构意味着你可以拥有大量可用的 Skills 而不会占满 Claude 的上下文窗口——Claude 只在需要时才访问需要的内容。

#### Skills vs. 子代理：怎么选？
- **用 Skills：** 当你希望任何 Claude 实例都能加载和使用某种能力时。Skills 就像培训教材——让 Claude 在所有对话中都更擅长特定任务。
- **用子代理：** 当你需要独立的、专门设计的助手来自主处理工作流时。子代理就像拥有独立上下文和工具权限的专职员工。
- **组合使用：** 当你希望子代理也具备专业知识时。例如，代码审查子代理可以加载 Skills 来遵循特定语言的最佳实践。

#### Skills vs. 提示词：怎么选？
- **用提示词：** 下达一次性指令、提供即时上下文或进行对话往复时。提示词是即时响应的、临时性的。
- **用 Skills：** 有需要反复使用的流程或专业知识时。Skills 是主动式的——Claude 知道何时该应用它们——且跨对话持久存在。
- **组合使用：** 提示词和 Skills 天然互补。用 Skills 提供基础专业知识，再用提示词为每次任务补充具体背景和微调。

#### Skills vs. 项目：怎么选？
- **用项目：** 需要为某项特定计划的所有对话提供背景知识和上下文时。项目提供的是始终加载的静态参考资料。
- **用 Skills：** 需要仅在相关时才激活的操作知识和可执行代码时。Skills 提供按需加载的动态专业能力，节省上下文空间。
- **关键区别：** 项目说的是"你需要知道这些"；Skills 说的是"要这样做"。项目提供你工作所需的知识库，Skills 提供随处可用的能力。

#### 子代理可以使用 Skills 吗？
可以。在 Claude Code 和 Agent SDK 中，子代理可以像主代理一样访问和使用 Skills。这创造了强大的组合——让专业化的子代理也能利用可移植的专业知识。

## 开始使用

准备好构建 Skills 了吗？以下是入门指南：

**Claude.ai 用户：**
- 在"设置" → "功能"中启用 Skills
- 在 [claude.ai/projects](https://claude.ai/projects) 创建你的第一个项目
- 尝试将项目知识与 Skills 结合，用于你的下一个分析任务

**API 开发者：**
- 在[文档](https://docs.anthropic.com/claude/docs/skills)中了解 Skills 端点
- 查看 [Skills Cookbook](https://github.com/anthropics/anthropic-cookbook)

**Claude Code 用户：**
- 通过插件市场安装 Skills
- 查看 [Skills Cookbook](https://github.com/anthropics/anthropic-cookbook)
