# Skills 详解：Skills 与提示词、项目、MCP 及子代理的对比

> 原文: https://claude.com/blog/skills-explained
> 作者: 
> 日期: 2026年3月20日

---

自从推出 **Skills**（技能）以来，人们对 Claude 代理化生态系统（agentic ecosystem）中各个组件如何协同工作表现出了浓厚的兴趣。

无论是在 **Claude Code** 中构建复杂的流水线，通过 API 创建企业级解决方案，还是在 **Claude.ai** 上提升生产力，了解应该在何时选择哪种工具，都能彻底改变你使用 Claude 的方式。

本指南将拆解每一个构建块，解释其适用场景，并向你展示如何将它们组合起来，以构建强大的代理化工作流（agentic workflows）。

## 了解你的代理化构建块

### 什么是 Skills？

**Skills** 是包含指令、脚本和资源的文件夹，Claude 会在处理相关任务时动态发现并加载它们。你可以将它们视为专门的“培训手册”，赋予 Claude 在特定领域的专业知识——从处理 Excel 表格到遵循组织的品牌准则。

**Skills 的工作原理：** 当 Claude 遇到任务时，它会扫描可用的 Skills 以寻找匹配项。Skills 采用**渐进式披露（progressive disclosure）**机制：首先加载元数据（约 100 个 token），提供足够的信息让 Claude 知道某个 Skill 是否相关。完整指令仅在需要时加载（少于 5k 个 token），捆绑的文件或脚本也仅按需加载。

**何时使用 Skills：** 当你需要 Claude 持续且高效地执行特定任务时，请选择 Skills。它们非常适合：

- **组织工作流：** 品牌准则、合规程序、文档模板。
- **领域专业知识：** Excel 公式、PDF 处理、数据分析。
- **个人偏好：** 笔记系统、编码模式、研究方法。

**示例：** 创建一个**品牌准则 Skill**，包含公司的调色板、字体规则和布局规范。当 Claude 创建演示文稿或文档时，它会自动应用这些标准，而无需你每次都重新解释。

了解更多关于 Skills 的信息，并查看我们不断增长的 [Skills 库](https://claude.com/skills)。

### 什么是提示词（Prompts）？

**提示词（Prompts）** 是你在对话过程中以自然语言向 Claude 提供的指令。它们是临时性的、对话式的且具有响应性——你根据当下的情况提供背景和方向。

**何时使用提示词：** 将提示词用于：

- **单次请求：** “总结这篇文章”。
- **对话微调：** “让语气更专业一点”。
- **即时上下文：** “分析这些数据并识别趋势”。
- **临时指令：** “将此格式化为无序列表”。

**示例：**

> 请对这段代码进行全面的安全审查。我关注以下内容：
>
> 1. 常见漏洞，包括：
> - 注入漏洞（SQL、命令、XSS 等）
> - 身份验证和授权问题
> - 敏感数据泄露
> - 安全配置错误
> - 失效的访问控制
> - 加密失败
> - 输入验证问题
> - 错误处理和日志记录问题
>
> 2. 对于发现的每个问题，请提供：
> - 严重程度（紧急/高/中/低）
> - 在代码中的位置（行号或函数名）
> - 解释为什么这是安全风险以及如何被利用
> - 具体的修复建议及代码示例
> - 防止类似问题的最佳实践指南
>
> 3. 代码上下文：[描述代码功能、语言/框架及运行环境 - 例如：“这是一个处理用户认证和支付数据的 Node.js REST API”]
>
> 4. 其他考虑因素：
> - 是否存在 OWASP Top 10 漏洞？
> - 代码是否遵循了 [特定框架/语言] 的安全最佳实践？
> - 是否存在已知漏洞的依赖项？
>
> 请按严重程度和潜在影响对发现结果进行排序。

**专业提示：** 提示词是你与 Claude 互动的主要方式，但它们不会跨对话持久存在。对于重复的工作流或专业知识，请考虑将提示词转化为 Skills 或项目指令。

**何时改用 Skill：** 如果你发现自己在多个对话中反复输入相同的提示词，那就该创建 Skill 了。将“使用 OWASP 标准审查此代码的安全性”或“按执行摘要、主要发现和建议的格式进行分析”等循环指令转化为 Skills。这可以免去你每次重新解释流程的麻烦，并确保执行的一致性。

查看我们的 [提示词库](https://claude.ai/prompts)、[提示词最佳实践](https://docs.anthropic.com/claude/docs/prompt-engineering) 或我们的 [智能提示词生成器](https://claude.ai/prompts) 来开始使用。

### 什么是项目（Projects）？

**项目（Projects）** 在所有 Claude 付费方案中均可用，是拥有独立聊天记录和知识库的自包含工作区。每个项目包含一个 200K 的上下文窗口，你可以上传文档、提供背景信息，并设置适用于该项目内所有对话的自定义指令。

**项目的工作原理：** 你上传到项目知识库的所有内容都可以在该项目的所有聊天中使用。Claude 会自动利用这些背景信息提供更明智、更相关的回答。当你的项目知识接近上下文限制时，Claude 会无缝启用 **检索增强生成（RAG）** 模式，将容量扩大至多 10 倍。

**何时使用项目：** 在以下情况下选择项目：

- **持久上下文：** 应该贯穿每一次对话的背景知识。
- **工作区组织：** 为不同的方案隔离不同的上下文。
- **团队协作：** 共享知识和对话记录（仅限团队和企业方案）。
- **自定义指令：** 项目特定的语气、视角或方法。

**示例：** 创建一个“第四季度产品发布”项目，包含市场调研、竞争对手分析和产品规格。该项目中的每一个聊天都能访问这些知识，而无需你重新上传或重新解释背景。

**何时改用 Skill：** 项目为 Claude 提供了针对特定工作主体的持久上下文——例如公司的代码库、某项研究计划、持续的客户互动。而 Skills 则是教 Claude **如何做** 某件事。一个项目可能包含产品发布的所有背景，而一个 Skill 可以教 Claude 团队的写作标准或代码审查流程。如果你发现自己在多个项目中复制相同的指令，那就是将其转化为 Skill 的信号。

了解更多关于 [项目](https://claude.ai/projects) 的信息。

### 什么是子代理（Subagents）？

**子代理（Subagents）** 是拥有独立上下文窗口、自定义系统提示词和特定工具权限的专用 AI 助手。子代理在 **Claude Code** 和 **Claude Agent SDK** 中可用，它们可以独立处理离散任务并将结果返回给主代理。

**子代理的工作原理：** 每个子代理都有自己的配置——你可以定义它的功能、解决问题的方法以及它可以访问的工具。Claude 会根据描述自动将任务委派给合适的子代理，或者你也可以显式请求特定的子代理。

**何时使用子代理：** 将子代理用于：

- **任务专业化：** 代码审查、测试生成、安全审计。
- **上下文管理：** 保持主对话专注，将专业工作分流出去。
- **并行处理：** 多个子代理可以同时处理不同方面。
- **工具限制：** 将特定子代理限制在安全操作内（例如只读访问）。

**示例：**
创建一个拥有读（Read）、Grep 和 Glob 工具访问权限，但没有写（Write）或编辑（Edit）权限的“代码审查”子代理。当你修改代码时，Claude 会自动委派该子代理进行质量和安全审查，而不会有意外修改代码的风险。

**何时改用 Skill：** 如果多个代理或对话需要相同的专业知识（如安全审查程序或数据分析方法），请创建一个 Skill，而不是将这些知识构建到单个子代理中。Skills 是可移植且可重复使用的，而子代理是为特定工作流量身定制的。使用 Skills 来教授任何代理都能应用的专业知识；当你需要具有特定工具权限和上下文隔离的独立任务执行时，请使用子代理。

了解更多关于 [子代理](https://docs.anthropic.com/claude/docs/subagents) 的信息。

### 什么是 MCP？

**MCP（模型上下文协议）** 在 AI 应用程序与你现有的工具和数据源之间创建了一个通用的连接层。

**模型上下文协议（Model Context Protocol, MCP）** 是一种开放标准，用于将 AI 助手连接到数据所在的外部系统——包括内容库、业务工具、数据库和开发环境。

**MCP 的工作原理：** MCP 提供了一种将 Claude 连接到工具和数据源的标准方式。无需为每个数据源构建自定义集成，只需针对单一协议构建即可。MCP 服务器公开数据和功能；MCP 客户端（如 Claude）连接到这些服务器。

**何时使用 MCP：** 当你需要 Claude 执行以下操作时，请选择 MCP：

- **访问外部数据：** Google Drive、Slack、GitHub、数据库。
- **使用业务工具：** CRM 系统、项目管理平台。
- **连接到开发环境：** 本地文件、IDE、版本控制。
- **与自定义系统集成：** 你的专有工具和数据源。

**示例：** 通过 MCP 将 Claude 连接到公司的 Google Drive。现在，Claude 可以搜索文档、读取文件并引用内部知识，而无需手动上传——连接是持久的且会自动更新。

**何时改用 Skill：** MCP 将 Claude 连接到数据；Skills 教 Claude **如何处理** 这些数据。如果你在解释如何使用工具或遵循程序——例如“查询我们的数据库时，始终先按日期范围过滤”或“使用这些特定公式格式化 Excel 报告”——这就是 Skill。如果你需要 Claude 首先访问数据库或 Excel 文件，这就是 MCP。将两者结合使用：MCP 负责连接，Skills 负责程序性知识。

了解更多关于 [MCP](https://modelcontextprotocol.io) 的信息，并查看有关 [如何构建 MCP 服务器](https://modelcontextprotocol.io/docs/quickstart) 的文档。

## 它们如何协同工作

当这些构建块组合在一起时，真正的力量才会显现。每一个构建块都有其独特用途，它们共同构成了复杂的代理化工作流。

### 对比：选择合适的工具

| 功能 | Skills | 提示词 (Prompts) | 项目 (Projects) | 子代理 (Subagents) | MCP |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **提供内容** | 程序性知识 | 即时指令 | 背景知识 | 任务委派 | 工具连接 |
| **持久性** | 跨对话 | 单个对话 | 项目内部 | 跨会话 | 持续连接 |
| **包含内容** | 指令 + 代码 + 资产 | 自然语言 | 文档 + 上下文 | 完整代理逻辑 | 工具定义 |
| **加载时机** | 动态加载，按需加载 | 每一轮对话 | 始终加载于项目中 | 被调用时 | 始终可用 |
| **可包含代码** | 是 | 否 | 否 | 是 | 是 |
| **最适合** | 专业技能 | 快速请求 | 集中上下文 | 特定任务 | 数据访问 |

### 代理化工作流示例：研究代理

让我们构建一个结合了多个构建块的全能研究代理。这个例子展示了如何组装并激活一个用于竞争分析的代理。

**第 1 步：设置你的项目**
创建一个“竞争情报”项目并上传：
- 行业报告和市场分析
- 竞争对手产品文档
- CRM 中的客户反馈
- 之前的研究摘要

添加项目指令：
> 请从我们产品战略的角度分析竞争对手。关注差异化机会和新兴市场趋势。以具体证据和可行建议的形式展示发现。

**第 2 步：通过 MCP 连接数据源**
启用以下 MCP 服务器：
- Google Drive（访问共享的研究文档）
- GitHub（审查竞争对手的开源仓库）
- Web Search（获取实时市场信息）

**第 3 步：创建专用 Skills**
创建一个“竞争分析”Skill：

> ### 我公司的 GDrive 导航 Skill
> #### 概览
> 针对 Meridian Tech 的 Google Drive 结构优化的搜索和检索策略。使用此 Skill 高效定位内部文档、研究成果和战略材料。
> #### 云端硬盘组织结构
> - `/Strategy & Planning/` - OKR、季度计划、董事会报告
> - `/Product/` - PRD、路线图、技术规格
> - `/Research/` - 市场研究、竞争情报、用户研究
> - `/Sales & Marketing/` - 案例研究、推介稿、营销材料
> #### 搜索最佳实践
> 1. **先广后窄** - 结合文件夹上下文和关键字。
> 2. **锁定文档所有者** - 从 Sales/ 文件夹获取销售材料，而不是根目录。
> 3. **检查时效性** - 优先考虑过去 6 个月内的文档以制定当前战略。

**第 4 步：配置子代理（仅限 Claude Code/SDK）**
创建专用子代理：

**市场研究子代理 (market-researcher)：**
> 角色：你是一名专注于竞争情报的市场研究分析师。
> 任务：识别权威来源，收集定量数据（市场份额等），分析定性见解（分析师观点等）。

**技术分析子代理 (technical-analyst)：**
> 角色：你是一名分析竞争对手技术选择的技术架构师。
> 任务：审查公共仓库和技术文档，评估架构模式和技术栈。

**第 5 步：激活你的研究代理**
现在，当你询问 Claude：“分析我们前三大竞争对手是如何定位其新的 AI 功能的，并识别我们可以利用的差距”时，会发生以下情况：

1. **项目上下文加载：** Claude 访问你上传的研究文档并遵循项目指令。
2. **MCP 连接激活：** Claude 搜索你的 Google Drive 获取最近的竞争对手简报，并抓取 GitHub 数据。
3. **Skills 介入：** “竞争分析”Skill 提供了分析框架。
4. **子代理执行（在 Claude Code 中）：** `market-researcher` 收集行业数据，而 `technical-analyst` 审查技术实现。
5. **提示词精炼：** 你提供对话式引导：“特别关注医疗保健领域的企业客户”。

**最终结果：** 一份全面的竞争分析报告，整合了多个数据源，遵循你的分析框架，利用了专业技能，并在整个研究过程中保持了上下文的一致性。

## 常见问题

#### Skills 的工作原理是什么？
Skills 使用**渐进式披露**来保持 Claude 的高效。在处理任务时，Claude 首先扫描 Skill 元数据（描述和摘要）以识别相关匹配。如果匹配，Claude 才会加载完整指令。最后，如果 Skill 包含可执行代码或参考文件，这些内容仅在需要时加载。
这种架构意味着你可以拥有许多可用的 Skills，而不会淹没 Claude 的上下文窗口。Claude 仅在需要的时间访问需要的内容。

#### Skills vs. 子代理：何时使用？
- **使用 Skills：** 当你想要任何 Claude 实例都能加载并使用的功能时。Skills 就像培训材料——它们让 Claude 在所有对话中都更擅长特定任务。
- **使用子代理：** 当你需要独立的、为特定目的设计的、能自主处理工作流的助手时。子代理就像拥有独立上下文和工具权限的专业员工。
- **结合使用：** 当你想要子代理拥有专业知识时。例如，一个代码审查子代理可以使用 Skills 来遵循特定语言的最佳实践。

#### Skills vs. 提示词：何时使用？
- **使用提示词：** 当你给出一次性指令、提供即时上下文或进行对话往复时。提示词是响应式的且临时性的。
- **使用 Skills：** 当你有需要反复使用的流程或专业知识时。Skills 是主动式的——Claude 知道何时应用它们——且在对话间持久存在。
- **结合使用：** 提示词和 Skills 自然互补。使用 Skills 提供基础专业知识，然后使用提示词为每个任务提供具体背景和精炼。

#### Skills vs. 项目：何时使用？
- **使用项目：** 当你需要背景知识和上下文来指导特定计划的所有对话时。项目提供始终加载的静态参考资料。
- **使用 Skills：** 当你需要仅在相关时激活的程序性知识和可执行代码时。Skills 提供按需加载的动态专业知识，节省上下文窗口。
- **关键区别：** 项目说的是“这是你需要知道的”；Skills 说的是“这是该怎么做的”。项目提供你工作的知识库；Skills 提供随处可用的能力。

#### 子代理可以使用 Skills 吗？
是的。在 Claude Code 和 Agent SDK 中，子代理可以像主代理一样访问并使用 Skills。这创造了强大的组合，让专业子代理可以利用可移植的专业知识。

## 开始使用

准备好构建 Skills 了吗？以下是开始的方法：

**Claude.ai 用户：**
- 在“设置”→“功能”中启用 Skills。
- 在 [claude.ai/projects](https://claude.ai/projects) 创建你的第一个项目。
- 尝试为你的下一个分析任务结合项目知识和 Skills。

**API 开发人员：**
- 在 [文档](https://docs.anthropic.com/claude/docs/skills) 中探索 Skills 端点。
- 查看我们的 [Skills Cookbook](https://github.com/anthropics/anthropic-cookbook)。

**Claude Code 用户：**
- 通过插件市场安装 Skills。
- 查看我们的 [Skills Cookbook](https://github.com/anthropics/anthropic-cookbook)。
